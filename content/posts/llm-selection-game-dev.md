# 大模型选型速查：从架构演进到游戏开发实践

## 一、架构演进要点

- **Multi-Head Latent Attention (MLA)**：KV Cache 先压缩后解压，长文本显存占用大幅下降，DeepSeek V2 起采用。
- **Mixture of Experts (MoE)**：上千专家参数，每次推理仅激活部分。DeepSeek V3 为 671B 总参 / 37B 激活，V4 达 1.6T。
- **流形约束超连接 (mHC)**：通过双随机矩阵稳定深层训练，支撑万亿级模型收敛。

## 三、游戏开发最佳实践

### 1. NPC 对话与剧情
- **方案**：核心剧情用 **Claude Sonnet 4.6**（英文）/ **Qwen3.6 Plus**（中文），路人 NPC 降级为 **DeepSeek V4 Flash**。
- **要点**：角色背景和历史写入 System Prompt，利用 1M 上下文 + 向量数据库保持人设一致性。

### 2. 代码辅助与关卡设计
- **方案**：复杂架构用 **Claude Opus 4.7** / **o3**，日常脚本用 **Qwen Coder**，本地离线用 **Llama4 Maverick**。
- **要点**：统一要求 JSON 结构化输出，直接对接游戏数据管线。

### 3. 多模态美术与音效
- **概念图**：**GPT-5.2** / **Gemini 3.0 Pro** 解析参考 → 生成 Prompt → Midjourney / SD。
- **语音**：**Qwen3.5 Omni** 端到端语音 MVP；正式配音用 Claude 生成带情感标记脚本 + **ElevenLabs**。
- **视频分析**：**Gemini 3.0 Pro** 自动分析录屏，检测 Bug。

### 4. 测试与本地化
- **自动化测试脚本**：**DeepSeek V4 Pro** 生成 Python 模拟脚本。
- **本地化初翻**：**Gemini 2.5 Flash**（便宜多语言），术语表固定在 System Prompt。
- **日志分析**：**Gemini 2.5 Pro**（2M 窗口）提取异常模式。

### 5. 分层部署与成本控制

| 层级 | 用途 | 推荐模型 | 策略 |
| :--- | :--- | :--- | :--- |
| **第一层 (实时)** | 玩家对话 | DeepSeek V4 Flash / GPT-4o-mini | 成本极低，快速响应 |
| **第二层 (异步)** | 主线剧情/代码 | Claude Sonnet/Opus | 离线批量生成 |
| **第三层 (审核)** | 内容安全 | 轻量模型 + 规则扫描 | 过滤不当内容 |
| **私有化** | 敏感数据 | Llama4 Scout 自部署 | 8×A100 支撑中等并发 |

**利用缓存**：重复系统提示开启 API 缓存，输入成本降低 **80%+**。

## 总结

大模型在游戏开发中的应用已从「尝鲜」走向「工业化」。核心原则是：
1. **分层部署**：实时交互用便宜模型，核心创意用旗舰模型。
2. **按需选模**：中文用 Qwen 系列，英文推理用 Claude/o3，成本敏感用 DeepSeek Flash。
3. **缓存与结构化输出**：开启 API 缓存，强制 JSON 输出对接管线，能大幅降低成本与对接难度。
