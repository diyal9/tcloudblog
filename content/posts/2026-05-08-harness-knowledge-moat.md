> 当 Harness Engineering 成为 2026 年最热门的 AI 工程话题，业界争论焦点集中在“该用多大的模型”还是“该搭多复杂的工作流”时，我们团队在落地实践中发现了一个被低估的事实—— **构建 Harness 工作流不是最终目的，私域和团队知识的沉淀才是真正的技术护城河** 。

### 一、工作流只是管道，知识才是活水

2025 年末至 2026 年初，AI 工程领域掀起了一场关于 **Harness Engineering** 的热烈讨论。Harness 的核心隐喻是“引导和约束 AI 模型的能力”，而非提升模型本身。

在深度实践的过程中，我们逐渐意识到一个更本质的问题：**工作流只是管道，知识才是流过管道的活水。** 模型会迭代，工具链会更新，工作流会重构。但你的团队在一个特定业务领域积累的 **领域模型、架构决策、最佳实践、已知陷阱** ——这些知识是永恒的。

### 二、Harness Engineering 三支柱与知识的位置

Harness 的核心要素可以归结为三个支柱：

| 上下文工程 | 架构约束 | 持续治理 |
| :--- | :--- | :--- |
| 长/短期记忆、知识检索注入 | Agent 编排模式、状态机设计 | 质量门禁、知识生命周期、自动衰减 |
| 渐进式披露、上下文防火墙 | 降级策略、安全边界 | 持续进化 |

**知识管理本身就是 Harness Engineering 的核心能力**，而不是附属品。

### 三、为什么知识沉淀比工作流更重要

1. **工作流是“可替换的”，知识是“可累积的”**：不管 Agent 调度模式怎么变，“广告预算扣减在高并发下会超扣”这条知识永远有价值。
2. **没有知识沉淀的工作流是“一次性”的**：每次从零开始推导架构决策，投入了工程成本却没有让工具链变聪明。
3. **知识是团队的“复利资产”**：分为散点型、因果型、时空型。当成百上千条经过验证的知识条目沉淀下来，新项目和新成员都能站在前人肩上。

### 四、知识分层架构：五层存储 × 五种类型 × 三级成熟度

在工程交付系统中，我们设计了一套三维正交的知识体系：

#### 4.1 五层存储架构
不同范围的知识有不同的共享边界和生命周期：
- **Layer 0-P（个人偏好）**：纯本地，不共享。
- **Layer 0-T（团队约定）**：代码规范、Commit 规范，团队层面的“宪法”。
- **Layer 1（技术知识）**：跨项目通用的技术经验。
- **Layer 2（业务知识）**：特定业务领域的领域模型、业务规则。
- **Layer 3（项目知识）**：仅在当前项目有意义的上下文。

#### 4.2 五种知识类型
遵循 MECE 原则分类：
- **model**：实体定义、数据结构。
- **decision**：技术选型、架构决策及理由。
- **guideline**：推荐做法或禁止做法。
- **pitfall**：已知风险、故障模式。
- **process**：业务流程、状态机。

#### 4.3 三级成熟度 + 自动衰减
知识不是写完就完了，它有生命周期：
- `draft`（新提取）→ `verified`（单项目验证）→ `proven`（成熟/可信赖）。
更关键的是 **自动衰减机制** —— proven 条目 12 个月未引用降级为 verified，verified 条目 6 个月未引用降级为 draft。避免过时知识误导 Agent。

### 五、团队知识库：独立 Git 仓库

我们做了一个关键的架构决策：**团队知识库是一个独立的 Git 仓库**，不寄生于任何业务项目。这确保了跨项目共享、生命周期独立和权限独立。

通过 **贡献暂存 + 异步合并** 模式，大多数情况（纯新增、证据追加）可以自动处理，只有真正的内容矛盾才需要 maintainer 介入。

### 六、工作流如何服务于知识沉淀

16 阶段状态机的每一个阶段都与知识的流动紧密关联：

1. **INIT 阶段（知识注入）**：启动时自动 `git pull` 团队知识库，注入 Agent 查询入口。
2. **执行中（知识消费）**：Agent 在决策点按需查询，比如查“有没有类似的架构决策”或“已知的反模式”。
3. **ARCHIVE 阶段（知识提取）**：完成后自动提取架构决策（decision）、踩过的坑（pitfall），符合条件的自动提升到 Layer 1 或 Layer 2。

### 七、突破人机交互瓶颈

工作流的流转依赖于人的在场。为了解决 Agent 在等待人工确认时工作流卡住的问题，我们引入了 **远程操控 + 跨设备接管** 能力：
- **状态持久化**：文件系统即状态机，随时从断点恢复。
- **异步审批**：人工确认节点设计为异步，Agent 提交产物后暂停等待，人类在任意设备审批后继续。
- **通知触达**：关键节点通过企微、飞书等渠道主动推送。

### 结语

好的 Harness 工程不仅要设计“Agent 怎么跑”，还要设计“人怎么随时参与”以及“知识怎么留下来”。
